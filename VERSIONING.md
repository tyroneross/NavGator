## npm provenance release prep

- `package.json` sets `publishConfig.provenance: true` and `publishConfig.access: public`.
- npmjs publishing uses `.github/workflows/publish.yml` with `id-token: write`, `npm publish --provenance --access public`, and a bypass-2FA `NPM_TOKEN` secret until npm Trusted Publisher OIDC is confirmed end-to-end.
- npm provenance is generated by the public GitHub Actions release.
- GitHub Packages publishing uses `GITHUB_TOKEN`, `packages: write`, and `npm.pkg.github.com`.
- The publish workflow must run the full test, typecheck, build, and packed-artifact verifier gates; release-only test exclusions are not allowed.
- The workflow builds and runs required Claude/Codex validation under the actual publish Node version, then packs once and publishes that same tarball to both registries with lifecycle scripts disabled so no post-verification rebuild can drift.
- `package.json.repository.url` must keep the exact GitHub repo casing (`tyroneross/NavGator`) for npm provenance matching.
- Public npm reports `@tyroneross/navgator@0.9.0` as `latest` as of 2026-06-08.
- After publish, verify with `npm view @tyroneross/navgator version dist-tags --json` and `npm audit signatures` in a consuming project.

## Release discipline (enforce before committing a version bump)

1. Bump `version` in `package.json` and `package-lock.json`
2. Synchronize `.claude-plugin/plugin.json`, `.codex-plugin/plugin.json`, and `.claude-plugin/marketplace.json`
3. Update the version stamp in `CLAUDE.md` if present
4. Update this file's `Current` section + add an entry to `Version history` below
5. Verify npm release readiness with the packed-artifact release verifier
6. Commit the release-prep files, push, then publish from the GitHub Actions release workflow so npm provenance is generated
7. After publish, verify npm, then update the external RossLabs marketplace entry and refresh local Claude/Codex plugin caches as separate authorized actions
