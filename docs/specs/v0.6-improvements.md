# NavGator v0.6 Improvement Backlog

From real-world validation on atomize-ai (v0.5.5).

## What NavGator Found Successfully
- Orphaned LLM chart generator (chart-spec-generator.ts — 714 lines, never called)
- Duplicate queue consumers (kg-summaries-groq → 2 competing workers)
- Dead infra (Heroku, Render detected but unused)
- Full pipeline trace: cron → route → service → database → queue → LLM
- 690 schema-to-code connections
- 1950 import connections (including @/ aliases)

## Improvements Needed (Priority Order)

### P0: Runtime fetch detection
Scan for `fetch('/api/...` patterns and create `frontend-calls-api` connections. This is the #1 gap for Next.js apps — pages fetch from API routes via HTTP at runtime, not imports. Without this, page → API → database traces are broken.

### P1: Backward trace
Follow import edges in reverse so `navgator trace X --direction backward` answers "who uses this module?" Currently returns 0 paths because it only follows forward-pointing edges.

### P1: Script classification
`scripts/` files classified as `production` — should be `script` or `dev-only`. Affects production filter accuracy.

### P1: LLM map accuracy
- Use primary provider (the `.create()` caller), not the tracing wrapper (LangSmith)
- Classify by file name keywords when purpose is UNKNOWN (chart-spec-generator → CHART-GENERATION)
- Deduplicate same-file multi-provider references

### P2: Anomaly detection
- Queues with 2+ consumers → warning in status/summary
- Files with identical connection patterns → potential duplicates
- Env vars defined but never read

### P2: Schema read/write distinction
Split `api-calls-db` into `schema-reads` (findMany, findUnique, count) and `schema-writes` (create, update, delete). Enables "which files write to Article?" queries.

### P3: File-level dead code
Detect files with 0 incoming import connections as potential dead code. Currently only flags component-level orphans.
